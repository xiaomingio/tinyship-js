# tinyship-js

English | [简体中文](README.zh-CN.md)

tinyship-js is a lightweight deployment tool for solo developers and small teams. It deploys Node.js / PM2 projects to servers you own.

It standardizes a common deployment flow: build the project locally, synchronize the files needed by each service to the matching host, install production dependencies on the server, reload the matching PM2 services, and load the matching env file at runtime.

## Installation

Install it in your project:

```bash
npm install @xiaomingio/tinyship-env
npm install -D @xiaomingio/tinyship
```

## Required Project Files

A project using TinyShip usually looks like this:

```text
project/
├── tinyship.config.yml             # Deployment config: hosts, services, and each host's file list
├── ecosystem.config.cjs            # PM2 service config
├── package.json                    # Project dependencies and scripts
├── package-lock.json               # Used by the server when installing production dependencies
├── src/                            # Source code
├── dist/                           # Locally built output, ignored by git
├── .env                            # Local development env file, ignored by git
├── .env.example                    # Committable env example
├── .env.prod.demo-service-one      # Production env for demo-service-one, ignored by git
├── .env.prod.demo-service-two      # Production env for demo-service-two, ignored by git
└── .env.prod.demo-service-three    # Production env for demo-service-three, ignored by git
```

`dist/` is generated by your own build command. TinyShip only handles synchronization and deployment.

## tinyship.config.yml

`tinyship.config.yml` describes deployment targets and file lists. SSH can use either `target` for an SSH alias / `user@host`, or split fields with `host`, `user`, `port`, and `identityFile`.

```yaml
hosts:
  demo-host-one:                 # Host name
    ssh:
      target: deploy@example.com # SSH alias or user@host
    appDir: /var/www/example     # Project directory on the server
    rsync:                       # File list deployed to this host
      - dist/                    # Local build output
      - package.json
      - package-lock.json
      - ecosystem.config.cjs
      - tinyship.config.yml
      - .env.prod.demo-service-one
      - .env.prod.demo-service-two

  demo-host-two:
    ssh:
      host: example.org             # SSH host
      user: deploy                  # Optional SSH user
      port: 2222                    # Optional SSH port
      identityFile: ~/.ssh/id_demo  # Optional private key path
    appDir: /var/www/example
    rsync:
      - dist/
      - package.json
      - package-lock.json
      - ecosystem.config.cjs
      - tinyship.config.yml
      - .env.prod.demo-service-three

services:
  demo-service-one:     # Service name, must match the PM2 app name
    host: demo-host-one # Deploys to hosts.demo-host-one
  demo-service-two:
    host: demo-host-one
  demo-service-three:
    host: demo-host-two
```

## ecosystem.config.cjs

PM2 service names must match the service names in `tinyship.config.yml`.

```js
module.exports = {
  apps: [
    {
      name: 'demo-service-one',
      script: 'dist/demo-service-one.js',
      env: {
        NODE_ENV: 'prod.demo-service-one',
      },
    },
    {
      name: 'demo-service-two',
      script: 'dist/demo-service-two.js',
      env: {
        NODE_ENV: 'prod.demo-service-two',
      },
    },
    {
      name: 'demo-service-three',
      script: 'dist/demo-service-three.js',
      env: {
        NODE_ENV: 'prod.demo-service-three',
      },
    },
  ],
};
```

## Loading env

Import `tinyship-env` at the very start of your application entry file to load the matching `.env` file:

```ts
import '@xiaomingio/tinyship-env/register';
```

TinyShip loads env files with these rules:

| Configuration | Loaded env file |
| --- | --- |
| `DOTENV_CONFIG_PATH` is set | File pointed to by `DOTENV_CONFIG_PATH` |
| `NODE_ENV` is `prod.demo-service-one` | `.env.prod.demo-service-one` |
| `NODE_ENV` is `prod.demo-service-two` | `.env.prod.demo-service-two` |
| Neither `DOTENV_CONFIG_PATH` nor `NODE_ENV` is set | `.env` |

## package.json

Add this to your project's root `package.json`:

```json5
{
  "scripts": {
    "build": "node scripts/build.mjs",                  // Run your project's own build flow
    "deploy:validate": "tinyship validate",             // Validate config, PM2 apps, env files, and local files
    "deploy:preflight": "tinyship preflight",           // Check SSH, remote tools, directory permissions, and local rsync
    "deploy:dry-run": "tinyship dry-run all",           // Preview deployment commands for all hosts
    "deploy:host-one": "tinyship deploy demo-host-one", // Deploy only demo-host-one
    "deploy:all": "tinyship deploy all"                 // Deploy all hosts
  }
}
```

## Server Requirements

Before the first deployment, prepare these on the server:

- SSH, with a local SSH key configured for passwordless login
- Node.js / npm / PM2
- rsync
- Writable project directory
- System dependencies required by your project

Prepare these locally:

- Node.js / npm
- rsync

Use `tinyship preflight` to check whether SSH, remote tools, and directory permissions meet the requirements.
